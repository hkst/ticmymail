Part‑2 provides a secure HTTP API for incident ingestion from Part‑1. It validates versioned payloads, applies config‑driven deduplication/merge, calls ServiceNow (ATLAS) via REST, and returns deterministic results with idempotency. It does not read mailboxes, call MS Teams, or send end‑user emails.
``` [1](https://livereadingac-my.sharepoint.com/personal/fk827241_student_reading_ac_uk/_layouts/15/Doc.aspx?sourcedoc=%7B57A0613E-9970-48AD-BE3E-721D4A20EE9E%7D&file=4_tmm_technicallowleveldesign_dev.docx&action=default&mobileredirect=true)

**6.2 API surface (skeleton)**
```text
POST /v1/incidents/ingest
Headers: Authorization: Bearer <token> | Idempotency-Key: <key> | X-Corr-Id: <uuid>
Body: { schema_version, event_type, message_id, thread_id, reported_at, subject, body, sender, attachments: [{name, size, type, uri?}], priority_hint?, correlation_hints: [...] }
Responses:
  200 { status: "merged"|"created"|"updated", sn_sys_id, correlation_id, action, now }
  400 { code: "INVALID_SCHEMA", ... }
  409 { code: "IDEMPOTENT_REPLAY", ... }
  502 { code: "SN_TIMEOUT", ... }
``` [1](https://livereadingac-my.sharepoint.com/personal/fk827241_student_reading_ac_uk/_layouts/15/Doc.aspx?sourcedoc=%7B57A0613E-9970-48AD-BE3E-721D4A20EE9E%7D&file=4_tmm_technicallowleveldesign_dev.docx&action=default&mobileredirect=true)

**6.3 Config versioning**
```text
At startup the service loads:
- /config/app.json (app settings, ServiceNow base URL)
- /config/schema/<version>.json (payload schema)
- /config/dedupe/<version>.json (rules)
The service exposes GET /v1/version showing the active versions. Config change policy: restart or hot-reload via SIGHUP (decide one).
``` [1](https://livereadingac-my.sharepoint.com/personal/fk827241_student_reading_ac_uk/_layouts/15/Doc.aspx?sourcedoc=%7B57A0613E-9970-48AD-BE3E-721D4A20EE9E%7D&file=4_tmm_technicallowleveldesign_dev.docx&action=default&mobileredirect=true)

**6.4 Logging schema**
```text
{ ts, level, corr_id, idem_key, schema_version, rules_version, route: "ingest|sn.create|sn.update", duration_ms, outcome, sn_sys_id?, error_code?, details? }
``` [1](https://livereadingac-my.sharepoint.com/personal/fk827241_student_reading_ac_uk/_layouts/15/Doc.aspx?sourcedoc=%7B57A0613E-9970-48AD-BE3E-721D4A20EE9E%7D&file=4_tmm_technicallowleveldesign_dev.docx&action=default&mobileredirect=true)

**6.5 Acceptance criteria (Part‑2‑specific)**
```text
- Accepts payloads matching current schema_version; rejects others with INVALID_SCHEMA (400).
- Processes a valid request end‑to‑end (validation→dedupe→ServiceNow call→response) in p95 ≤ 5s under N=XX RPS.
- Applies idempotency: same Idempotency-Key returns 200 with same sn_sys_id and outcome without duplicate side effects.
- Emits structured logs for 100% of requests with corr_id and idem_key.
- Exposes /health (liveness/readiness) and /version.
``` [1](https://livereadingac-my.sharepoint.com/personal/fk827241_student_reading_ac_uk/_layouts/15/Doc.aspx?sourcedoc=%7B57A0613E-9970-48AD-BE3E-721D4A20EE9E%7D&file=4_tmm_technicallowleveldesign_dev.docx&action=default&mobileredirect=true)
